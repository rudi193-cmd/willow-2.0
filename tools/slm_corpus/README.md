# slm_corpus — lane-4 SLM training corpus pipeline

b17: SLMC1 · ΔΣ=42

Builds the training corpus for a small language model to run Willow's
**lane-4 workhorse tasks** — the bounded jobs currently sent to local
Ollama models (mistral:7b / llama3.2:3b): summarize, classify, extract,
tension, intake routing, citation corroboration, stop summaries, dream
synthesis, KB drift auditing.

Every template in `templates.py` mirrors a production call site exactly
(provenance in its docstring), so a model fine-tuned on this corpus drops
into the existing code with no prompt changes.

## Privacy

The pipeline code is public. The corpus is not. All data is written to
`WILLOW_SLM_CORPUS_DIR` (default `<willow_home>/slm-corpus/`) which lives
outside this repository. Harvest redacts secret-shaped strings and skips
atoms whose `sensitivity` is not `open` unless `--include-sensitive`.

## Flow

```
harvest.py    real fleet inputs → inputs.jsonl
                sources: knowledge atoms (+semantic-neighbor pairs),
                intake JSONL, session transcripts, git history, Grove
gold/*.jsonl  gold outputs, generated by a frontier model
                {"id": <input id>, "output": "<assistant text>"}
baseline.py   local-model outputs for the same inputs → baseline/*.jsonl
assemble.py   validate + canonicalize → sft_train/sft_val.jsonl, dpo.jsonl
```

Gold outputs are validated against each task's strict production contract
(`templates.validate_output`) and canonicalized (bare JSON, no fences)
before entering the SFT set. Baseline outputs that fail the contract while
gold passes become DPO `rejected` candidates.

## Typical run (via Kart)

```bash
python3 tools/slm_corpus/harvest.py --task all --limit 200
# ... generate gold outputs into <corpus>/gold/ ...
python3 tools/slm_corpus/baseline.py --limit 500
python3 tools/slm_corpus/assemble.py
```

Output format is standard chat JSONL (`messages` array), compatible with
axolotl / unsloth / LLaMA-Factory SFT and DPO loaders.
